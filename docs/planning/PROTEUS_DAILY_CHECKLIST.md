# Proteus Daily Checklist

Use this once per day. Keep it to 10-20 minutes of planning max.

- [ ] Pick exactly one ticket from `docs/planning/PROTEUS_TICKETS.md`
- [ ] Write 3-bullet definition of done in `docs/planning/PROTEUS_BUILD_LOG.md`
- [ ] Write the test(s) you will run before coding
- [ ] Implement only that ticket scope
- [ ] Run tests and note failures/risks
- [ ] Log what changed and choose next ticket
